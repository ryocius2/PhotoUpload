Wedding Photo Upload - Setup TODO
==================================

[ ] 1. TEST LOCALLY
      - Double-click run_local.bat
      - Open http://localhost:5000/?table=7 in your phone or browser
      - Try taking a photo and uploading
      - Check that photos/ folder gets the image

[ ] 2. SET UP RASPBERRY PI
      - Flash Raspberry Pi OS Lite onto SD card
      - SSH in and clone/copy this project over
      - pip install -r requirements.txt
      - pip install qrcode[pil]
      - cp .env.example .env and set a real SECRET_KEY

[ ] 3. CLOUDFLARE TUNNEL (public HTTPS access)
      - Install cloudflared on Pi:
          curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm64.deb -o cloudflared.deb
          sudo dpkg -i cloudflared.deb
      - cloudflared tunnel login
      - cloudflared tunnel create wedding-photos
      - cloudflared tunnel route dns wedding-photos photos.YOURDOMAIN.com
      - cloudflared tunnel run --url http://localhost:5000 wedding-photos
      - Note: you need a domain on Cloudflare (free plan works)

[ ] 4. PRODUCTION SERVER
      - pip install gunicorn
      - Run with: gunicorn -w 4 -b 0.0.0.0:5000 app:app
      - Optional: set up systemd service so it starts on boot

[ ] 5. ONEDRIVE BACKUP
      - sudo apt install rclone
      - rclone config  (add "onedrive" remote, follow browser auth)
      - Test: rclone ls onedrive:
      - chmod +x sync_onedrive.sh
      - crontab -e -> add: */10 * * * * /home/pi/PhotoUpload/sync_onedrive.sh

[ ] 6. GENERATE QR CODES
      - python generate_qr.py --url https://photos.YOURDOMAIN.com --tables 15
      - Print the PNGs from qr_codes/ folder
      - Put them in table card holders or tent stands

[ ] 7. DAY-OF CHECKLIST
      - Pi plugged in and powered on at home
      - Cloudflare tunnel running
      - Test QR code from phone at venue (cell data, not home wifi)
      - Check OneDrive sync is working
      - Bring a USB backup drive just in case
